# Pricing & Proposal Phases

## PRICING

### Three modes — the learner chooses

1. **Manual**: learner enters unit costs and labor rates. Set up the grid and
   accept their numbers without second-guessing (flag only arithmetic issues).
2. **Auto-fill reference**: search current web pricing, decompose ranges into
   per-line material + labor references, and fill the grid — with EVERY number
   clearly flagged as a placeholder to review and edit. Web pricing is
   regional, dated, and homeowner-retail-skewed; say so.
3. **Hybrid**: auto-fill, learner overrides line by line.

### Rules regardless of mode

- **Code computes all extensions and totals.** Models write words around
  figures; they never restate or re-derive a computed figure.
- **Sanity-check the total** against current market ranges for the system
  type and job type (new construction vs. replacement changes the range
  materially — new ductwork is a large adder). Present the cross-check to the
  learner; it's also the lesson in defending a bid.
- **Overhead & profit is the learner's business decision.** Offer a
  placeholder (e.g., 15%) labeled as such.
- **Scope items price too**: load calcs (Manual J/S/D) if contractor's
  responsibility, condensate assemblies, line set/charge/startup, sealing and
  testing, permits. These came from the notes — they are line items, not
  footnotes.
- Equipment lines carry the most dollars and variance; recommend real
  supplier quotes over any web reference for a live bid.

## PROPOSAL

The deliverable a GC receives. Structure:

1. **Header/info block**: from/to, license, project, referenced documents,
   validity period.
2. **Scope of work narrative**: complete-and-functional framing when the
   notes contain a complete-system clause; name the code basis; name design
   responsibilities (Manual J/S/D) if contractor's.
3. **Base bid table**: the takeoff rolled up by section (equipment, air
   distribution, ductwork, design/closeout), quantities, amounts, O&P line,
   total. Numbers come verbatim from the code-computed takeoff.
4. **Inclusions**: accessory scope from schedule remarks and notes (filters,
   aux heat, dampers/timers/caps, supports, condensate, tie-downs, sealing,
   calcs, permit). Everything mined in orientation Step 4/5 lands here or in
   qualifications.
5. **Exclusions**: cross-trade catches, stated with their source ("electrical
   disconnects by others per E1.0 Note 16"; plumbing scope per P sheets;
   painting/patching; structural beyond equipment pads).
6. **Qualifications**: every ambiguity the workflow surfaced, converted to a
   qualification instead of a silent assumption. Standard set from piloting:
   - Drawings schematic; sizing by contractor's Manual J/S; pricing assumes a
     stated nominal tonnage, adjusted if the calc requires materially more.
   - Cross-trade supply questions (e.g., combination light/exhaust-fan
     fixtures appearing on electrical sheets) — confirm responsibility with
     the GC prior to award.
   - Duct quantities measured from schematic plans at stated scale with the
     stated allowance; final routing per field conditions and Manual D.
   - Pricing validity window.
7. **Signature block** for both parties.

### Traceability standard

Every scope statement, inclusion, exclusion, and qualification traces to a
sheet, note, schedule remark, or verified count. The test: if the GC
challenges any line, the answer is a document reference. If a statement has
no source, it doesn't go in the proposal.

### Output format

Produce the proposal as a real PDF (reportlab), professional layout, with
bracketed placeholders for the contractor's identity fields. The priced table
must match the locked takeoff exactly — regenerate from the data, never
retype.
